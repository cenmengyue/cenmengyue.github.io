## 1. 适合“先搜再写”的两段式提示词

这个最适合实际工作流。

### 第一段：搜索筛选阶段

```text
请作为多模态文献研究助理，围绕主题“[你的主题]”进行文献搜索。

目标：
- 找到近 2-3 年最相关、最有代表性的论文
- 补充少量经典基础论文
- 去重并筛选出最值得读的 10 篇

请输出：
1. 检索关键词（中英文）
2. 检索范围与来源
3. 候选论文列表（题目、年份、会议、链接）
4. 每篇论文的入选理由
5. 推荐阅读顺序
6. 按“综述类 / 基础方法 / 最新方法 / 与我主题最相关”分类整理

注意：
- 优先多模态、视觉语言、MLLM、视频、多图推理、segmentation、reasoning 等相关工作
- 不要编造链接
- 如果某篇论文相关性一般，请明确标注
```

### 第二段：生成笔记阶段

```text
现在请根据上一步筛选出的论文，为每篇论文生成独立的 Markdown 笔记。

每篇笔记必须包含：
- 标题
- 作者
- 会议/年份
- 论文链接
- 项目页/代码
- 研究问题
- 摘要简述
- 主要方法
- 方法 pipeline（原图位置/图链接 + Mermaid 流程图 + 文字解释）
- 核心创新点
- 实验与结果
- 优势
- 不足与局限
- 未来改进方向
- 我的阅读备注
- 一句话总结

要求：
- 所有内容尽量中文
- 不确定的信息必须标注
- 不要泛泛而谈，尤其是“不足”和“未来方向”
- 输出格式统一，适合直接保存为 `.md`
- 一次先输出 1 篇，等我说“继续”再输出下一篇
```

## 2. 如果你想让它“更像研究员”，可以加这一段

你可以附加到主提示词最后：

```text
请以“研究员做文献笔记”的标准来写，而不是只做摘要搬运。尤其要做到：

- 区分“论文明确写出”与“你基于内容推断”
- 对方法拆解到模块级别，而不是只重复摘要
- 对创新点要说明“相对谁创新”
- 对不足要具体到：
  - 任务设定是否受限
  - 数据集偏差
  - 泛化问题
  - 推理成本
  - 训练复杂度
  - 对标注/外部模型/额外数据的依赖
  - 是否真的具有可扩展性
- 对未来方向要和当前局限一一对应
- 如果一篇论文的方法图很关键，请优先解释图中每个模块的输入输出
```

## 3. 在实际使用时这样发

你在网页版里可以这样分三轮用。

### 第 1 轮：设定规则

把“主提示词”发过去。

### 第 2 轮：给主题

```text
研究主题：多模态大模型中的视觉定位与 reasoning segmentation
重点关注：
- visual grounding
- referring segmentation
- reasoning segmentation
- MLLM with segmentation
- RL/GRPO for perception
时间范围：2024-2026
优先会议：CVPR, ICCV, ECCV, NeurIPS, ICLR, arXiv
请先给我候选论文列表，不要直接写 markdown。
```

### 第 3 轮：开始输出笔记

```text
从最相关的 5 篇开始。
每次只输出 1 篇 markdown，方便我检查。
```

## 4. 场景专用版

```text
你现在是我的多模态研究助理，帮助我做论文搜索、阅读整理和笔记沉淀。我的目标不是只看摘要，而是形成可长期积累的研究笔记库。

请围绕我给定的多模态主题，完成以下工作：

1. 检索相关论文
- 优先近 2-3 年
- 补充少量奠基论文
- 优先顶会/高质量 arXiv
- 优先有代码、项目页、清晰方法图的论文

2. 筛选并排序
- 按相关性、代表性、创新性排序
- 去掉明显重复、边缘相关、实验不足的论文
- 先输出候选列表，不要直接展开长篇总结

3. 对每篇论文输出独立 Markdown 笔记
笔记必须包含：
- 论文标题
- 作者
- 会议/年份
- 论文链接
- 项目页/代码
- 研究问题
- 摘要中文总结
- 主要方法拆解
- 方法 pipeline
- 核心创新点
- 实验结果
- 优势
- 不足之处
- 未来改进方向
- 我的阅读备注
- 一句话总结

4. 关于 pipeline 的要求
- 优先给出论文原图所在位置（Figure 编号、章节）
- 若能提供图片链接则提供
- 若不能直接提供图片，则使用 Mermaid 画出简化流程图
- 再补充模块级文字解释（输入、模块、输出、训练、推理）

5. 关于写作质量的要求
- 用中文写，术语保留英文
- 不要只复述 abstract
- 不要编造论文中未明确写出的信息
- 对创新点要说明相较于哪些已有方法的改进
- 对不足要具体，不要空泛
- 对未来方向要和论文局限对应
- 对不确定内容标注“论文未明确说明”或“基于内容推断”

6. 输出方式
- 先输出候选论文列表 + 入选理由 + 推荐阅读顺序
- 我确认后，你再逐篇输出 Markdown
- 每次只输出一篇，便于我检查与保存
- Markdown 格式适合直接保存为 `.md` 文件
```

## 5. 额外建议

```text
请优先保证准确性和结构化，不要为了完整而编造细节。
```

```text
若某篇论文你无法确认图示、结果或代码链接，请明确告诉我，不要猜测。
```
